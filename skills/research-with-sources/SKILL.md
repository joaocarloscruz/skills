# Research with Sources

## Frame the question

1. Translate the request into a concrete question, decision, or comparison.
2. Record constraints such as date, jurisdiction, platform, version, audience, and desired depth.
3. Identify which claims may have changed and require current verification.

## Gather evidence

1. Prefer the source that owns the fact: official documentation, specifications, source code, first-party datasets, filings, or original papers.
2. Use secondary sources for discovery or interpretation, then trace important claims back to primary evidence.
3. Check publication dates and the date an event actually occurred.
4. Seek independent corroboration for consequential or disputed claims.
5. Record the direct URL and the exact claim each source supports.

Do not pad the source list. A few strong, directly relevant sources are better than many weak ones.

## Synthesize

1. Lead with the answer or decision-relevant conclusion.
2. Distinguish sourced facts, source disagreements, and your own inferences.
3. Cite claims near the text they support; link to the specific page rather than a search result.
4. Note material uncertainty, missing data, version caveats, and assumptions.
5. Respect copyright: quote only what is necessary and otherwise paraphrase.

When saving a brief, use the repository's existing documentation location and naming convention. Do not create a new file unless the user asked for an artifact or the workflow clearly requires one.
